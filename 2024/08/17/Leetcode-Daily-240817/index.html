事前分析

Recursion

Alg

用 Recursion 的做法，我們可以定義 $f(i,j)$ 是考慮第 1~i 列，選擇了第 i 列第 j 行的元素，所能得到的最大點數。

$$f(i,j)= \\
\begin{cases}
\mathop{\max}_{1 \leq k \leq m}f(i-1,k)+points[i][j]-|j-k| & i>1 \\
points[i][j] & i=1 \\
\end{cases}
$$

然後答案就會是 $\mathop{\max}_{1\leq k \leq m}f(n,k)$

Time Complexity

$O(m^n)$ (每一個問題延伸出 $m$ 個子問題，共有 $n-1$ 層這樣的結構，共要解 $m$ 個這樣的問題)

因為 $m$, $n$ 量級用此複雜度的演算法絕對跑不完，故直接否決 Recursion 做法。

Dynamic Programming

Observation

從遞迴的使用不難發覺，每個問題可以由回答更小的子問題來求得答案。
原問題得到最大的點數時，子問題也會是最大的點數。
子問題間彼此不影響，但有重複的問題要解。
因此可以使用 dp 來解決此題。

Alg

定義 dp[i][j] 就是和 $f(i,j)$ 一樣的問題。
解 dp[i][j] 的轉移式也和遞迴版本完全一樣，只是 dp[i-1][k] 這些子問題的答案已經被算好，並記錄起來了，因此可以獲取更好的複雜度。

Time Complexity

第一個 for 是 $O(m)$；最後的 max 是 $O(m)$；中間的三層迴圈是 $O(nm^2)$。故總體時間複雜度為 $O(nm^2)$。

原本以為到這裡就可以開心結束此題，但事情沒有這麼簡單。
我看到了 $n$,$m$ 的範圍，驚覺 $O(nm^2)$ 的時間複雜度依然不足以通過此題，實際一送也得到 TLE 的結果。

最終解法

Observation

因為題目有給一個 $nm$ 一起的範圍限制，我猜測要把複雜度壓到 $O(nm)$ 才行。

我觀察了一下 dp 的式子

$$dp[i][j] =\mathop{\max}_{1 \leq k \leq m}dp[i-1][k]+points[i][j]-|j-k|$$

$$=\max\{\mathop{\max}_{1 \leq k \leq j} dp[i-1][k] + k + points[i][j] - j,$$

$$\mathop{\max}_{j < p \leq m} dp[i-1][p] - p + points[i][j] + j\}$$

$$=\max\{\max\{dp[i-1][k]+k\}-j,\max\{dp[i-1][p]-p\}+j\} + points[i][j]$$

最後一行省略了 max 的下限。

我們把最一開始的一個 max 拆成了兩個 max，並且這兩個 max 內的內容和 j 獨立。

這麼一來我們可以不用在跑 i, j 的迴圈中不斷地再跑 k 的迴圈去運算。而可以在跑 i 時先把我們需要的兩個 max 計算完畢並記錄起來，跑 j 時就可以常數時間拿到最大值。
